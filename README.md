## Repository Organization Best Practices

- Use clear, descriptive names: `domain-purpose-tech` (e.g., `product-list-with-cart-main`)
- Keep a consistent structure: `src/`, `public/`, `assets/`, `README.md`
- Add topics for discoverability: `javascript`, `typescript`, `react`, `css`, `php`
- Write a helpful README with: problem overview, tech stack, setup, scripts, and screenshots
- Include a license when applicable and basic contribution guidelines
